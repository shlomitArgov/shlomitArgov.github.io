Linux user types

root (system admin) - not bound to the permissions mechanism
user - everyone other than root

User definitions

/etc/passwd
  • Can only be edited by root
  • Contains a single line for each user, fields are separated by ':' 
  • Sample format: smithj:x:561:561:Joe Smith:/home/smithj:/bin/bash
  • Fields are:
    •  username
    • 'x' (originally the user password was placed here, now its hash is stored in /etc/shadow)
    • numeric user id (not unique, used for permissions, root user has value of 0)
    • numeric group id
    • comment
    • home directory
    • default shell

/etc/shadow
  • Can only by viewed or edited by root
  • Contains a single line for each user
  • Sample format: smithj:Ep6mckrOLChF.:10063:0:99999:7::: 
  • Fields are:
    • username
    • encoded password (during login, if this string is generated with the user's password, then the login succeeds) - This field can be left empty which means the user can login without a password. If this field contains '*' it means the account has been disabled.
    • ...
/etc/sudoers
Defines which users can perform what actions and under which user (via sudo).
